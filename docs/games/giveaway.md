# Розыгрыш

Многоместный розыгрыш с призовыми местами, управляемый администратором.

## Команда

```
/giveaway <приз1> [приз2] [приз3] ... [время]
```

Примеры:

```
/giveaway 500 100 50 30m     # 3 места, розыгрыш через 30 минут
/giveaway 1000 2h            # 1 место, розыгрыш через 2 часа
/giveaway 200 50             # 2 места, без таймера (завершить вручную)
```

!!! note "Только для администраторов"
    Команда доступна только пользователям из списка `admin.users` в `configs/config.yaml`.

## Как это работает

1. Администратор создаёт розыгрыш с призами и опциональным таймером.
2. Бот публикует объявление с кнопкой **🎟 Участвовать**.
3. Любой участник чата нажимает кнопку и попадает в список.
4. По истечении таймера (или вручную) бот случайным образом выбирает победителей и начисляет призы.

## Призовые места

Призы передаются позиционно: первый аргумент — 1-е место, второй — 2-е и т.д.

```
/giveaway 500 100 50
```

| Место | Приз |
|-------|------|
| 🥇 1-е | 500 очков |
| 🥈 2-е | 100 очков |
| 🥉 3-е | 50 очков |

Если участников меньше, чем призовых мест — заполняются только доступные места.

## Время ожидания

Формат: `<число>m` (минуты) или `<число>h` (часы).

| Пример | Время |
|--------|-------|
| `30m` | 30 минут |
| `2h` | 2 часа |
| *(не указан)* | Только ручное завершение |

Если таймер не указан — розыгрыш остаётся активным до тех пор, пока администратор его не завершит (через механизм фонового цикла или вручную).

## Технические детали

- Состояние хранится в **PostgreSQL** (`giveaways`, `giveaway_participants`)
- Фоновый цикл (`giveaway_loop`) проверяет завершённые розыгрыши каждые 60 секунд
- Победители выбираются случайно (`random.sample`) без повторений
- Логика в `bot/application/giveaway_service.py`
- Хендлер в `bot/presentation/handlers/giveaway.py`
